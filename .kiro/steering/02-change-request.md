# Phase 2: Change Request (CR)

Generate a formal CR from parsed requirements. Save to `docs/01-change-request/CR-[ID]-v1.0.md`.

## Sections

1. Executive Summary — 2-3 paragraphs, non-technical audience
2. Scope — In Scope (bulleted), Out of Scope (bulleted), Impacted Systems table
3. Requirements Traceability — Table mapping REQ IDs to acceptance criteria
4. Business Rules — BR-XXX IDs, traced to REQs
5. Constraints & Dependencies — Technical, business, external
6. Assumptions — Carried from Phase 1, updated
7. Risks — Risk ID, description, probability, impact, mitigation
8. Approval Block — Checklist for user to confirm, instruction to reply "Approved"

## GATE: Do NOT proceed without explicit user approval.
